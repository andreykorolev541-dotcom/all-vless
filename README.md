# VLESS Subscription Aggregator (IPv4 + IPv6)

Автоматический агрегатор VLESS-конфигураций из популярных открытых источников. Скрипт собирает публичные прокси-серверы, автоматически декодирует Base64 данные, валидирует структуру ссылок (поддерживая IPv4, IPv6 и доменные имена) и очищает итоговый список от дубликатов.

Обновляется каждые 6 часов автоматически.

  `https://raw.githubusercontent.com/Rageru01/white-list/main/configs/vless_plain.txt`
  *(Ссылки `vless://` идут построчно в открытом виде)*

## 📱 Как добавить подписку в клиент

### v2rayNG / v2rayN (Android / Windows):
1. Откройте меню подписок (Subscription settings).
2. Нажмите **Добавить (+)**.
3. Введите любое имя и вставьте скопированную ссылку на `vless_base64.txt`.
4. Сохраните и нажмите **Обновить подписки**.

### Hiddify (Android / iOS / Desktop):
1. Перейдите в раздел **Новый профиль** (или Настройки -> Профили).
2. Нажмите **Добавить подписку по URL**.
3. Вставьте ссылку на `vless_base64.txt` и нажмите **Сохранить**.

### Nekobox / Sing-box:
1. Зайдите в группы/профили.
2. Создайте новую группу с типом **Subscription** (Подписка).
3. В поле URL вставьте ссылку на файл `vless_base64.txt` и обновите группу.

## 🛠️ Как это работает

1. **Сбор данных:** Скрипт по расписанию обходит доверенные публичные репозитории, где публикуются бесплатные VLESS-конфигурации.
2. **Декодирование и парсинг:** Скрипт умеет "на лету" расшифровывать Base64-потоки и извлекать из них только валидные строки протокола `vless://`.
3. **Поддержка IPv6:** Регулярное выражение корректно распознает сервера, работающие на протоколе IPv6 (в квадратных скобках `[...]`), сохраняя их наряду с привычными IPv4 и доменами.
4. **Удаление дубликатов:** Все повторяющиеся конфигурации серверов автоматически склеиваются, чтобы ваш клиент не загружал одинаковые прокси.

## 🔄 Автоматическое обновление

Сборщик работает полностью автономно через **GitHub Actions**:
* **По расписанию:** каждые 6 часов.
* **Вручную:** Если вам нужно обновить список прямо сейчас, перейдите во вкладку **Actions** → выберите workflow **Update Working Configs** → нажмите **Run workflow**.

## 📂 Структура проекта

* `scripts/check_configs.py` — Главный скрипт парсера и валидатора.
* `configs/vless_plain.txt` — Собраные конфигурации в текстовом виде (построчно).
* `configs/vless_base64.txt` — Итоговая подписка, закодированная в Base64 для импорта в приложения.
* `configs/stats.json` — Статистика и метаданные последнего запуска (время, количество рабочих конфигураций).
